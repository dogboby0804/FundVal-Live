# Fundval 问题排查手册

## 1. 注册/登录问题

### 1.1 提示"注册失败"或"无法连接服务器"

**排查步骤**：

1. **确认客户端类型**
   - Web 浏览器：检查 URL 是否正确
   - Desktop (Tauri)：检查设置中的服务器地址
   - Android (Capacitor)：检查应用设置中的服务器配置

2. **验证服务器连通性**
   ```bash
   # 测试后端 API 是否可访问
   curl http://your-server:21345/api/health
   ```

3. **检查注册功能是否开启**
   - 访问 `/initialize` 页面查看系统配置
   - 如果注册功能关闭，联系管理员开启
   - 开启后需要**重启后端服务**

4. **测试环境**
   - 可使用测试服务器：`https://fund.jasxu.dpdns.org/`
   - 测试环境不保证数据持久化，请勿使用真实数据

### 1.2 登录后提示"Bootstrap Key 未初始化"

**原因**：系统首次启动未完成初始化

**解决方案**：

重启backend

```bash
# Docker 部署
docker-compose logs backend | grep 'BOOTSTRAP KEY'

# 手动部署
./start.sh  # 日志中会显示 BOOTSTRAP KEY
```

访问 `http://your-server:21345/initialize`，填入 Bootstrap Key 完成初始化。

## 2. 基金数据问题

### 2.1 没有基金数据 / 基金列表为空

**排查步骤**：

1. **手动触发同步**
   ```bash
   # Docker 部署
   docker-compose exec backend python manage.py sync_funds

   # 手动部署
   cd backend && uv run python manage.py sync_funds
   ```

2. **检查数据源连通性**
   ```bash
   # 测试东方财富接口
   curl -v http://fundgz.1234567.com.cn/js/001237.js

   # 应返回类似：jsonpgz({"fundcode":"001237",...})
   ```

3. **查看同步日志**
   ```bash
   # Docker 部署
   docker-compose logs backend | grep sync_funds

   # 手动部署
   tail -f backend/logs/django.log
   ```

4. **Docker 自动同步说明**
   - `entrypoint.sh` 会在首次启动时自动执行 `sync_funds --if-empty`
   - 如果数据库已有数据，不会重复同步
   - 强制重新同步需手动执行命令

### 2.2 估值数据不更新

**可能原因**：

1. **Celery Worker 未启动**
   ```bash
   # Docker 部署
   docker-compose ps | grep worker

   # 手动部署
   ps aux | grep celery
   ```

2. **Redis 连接失败**
   ```bash
   # 测试 Redis 连接
   docker-compose exec backend python -c "import redis; r=redis.Redis(host='redis'); print(r.ping())"
   ```

3. **定时任务未触发**
   - 估值更新时间：交易日 9:30-15:00 每分钟
   - 净值更新时间：每天 18:30 / 21:30 / 23:00

   ```bash
   # 查看 Celery Beat 日志
   docker-compose logs beat | tail -50
   ```

## 3. 持仓计算问题

### 3.1 清仓后持仓列表仍显示全 0 记录

**原因**：旧版本 bug，清仓后未删除 Position 记录

**解决方案**（v1.x.x 已修复）：

1. **触发重算**
   - 删除该基金的任意一条流水记录
   - Signal 会自动触发 `recalculate_position`
   - 重新添加流水即可

2. **批量清理**
   ```bash
   # 进入 Django shell
   docker-compose exec backend python manage.py shell

   # 执行清理
   from api.models import Position
   Position.objects.filter(holding_share=0).delete()
   ```

### 3.2 持仓成本计算不准确

**排查步骤**：

1. **检查流水记录**
   - 确认所有买入/卖出操作的净值、份额、金额是否正确
   - 卖出时成本按比例减少（FIFO 加权平均）

2. **手动触发重算**
   ```bash
   docker-compose exec backend python manage.py shell

   from api.services import recalculate_position
   recalculate_position('account_id', 'fund_id')
   ```

3. **查看计算逻辑**
   - 代码位置：`backend/api/services/__init__.py`
   - 算法：加权平均成本法

## 4. 数据源问题

### 4.1 养基宝登录失败

**排查步骤**：

1. **检查二维码是否过期**
   - 二维码有效期 2 分钟
   - 超时需重新获取

2. **检查网络连接**
   ```bash
   curl -v https://www.yangjibao.com/api/v1/user/info
   ```

3. **Token 失效**
   - 养基宝 Token 有效期约 30 天
   - 失效后需重新扫码登录

注意：ADGuard插件可能拦截养基宝二维码，可以尝试更换浏览器或者临时禁用该插件

### 4.2 东方财富数据获取失败

**可能原因**：

1. **接口限流**
   - 东方财富有请求频率限制
   - 建议间隔 1 秒以上

2. **基金代码不存在**
   - 确认基金代码是否正确
   - 部分基金可能未在东方财富上线

3. **网络问题**
   ```bash
   # 测试连通性
   curl -v http://fundgz.1234567.com.cn/js/001237.js
   ```

## 5. Docker 部署问题

### 5.1 容器启动失败

**排查步骤**：

1. **查看容器日志**
   ```bash
   docker-compose logs backend
   docker-compose logs frontend
   docker-compose logs worker
   ```

2. **检查端口占用**
   ```bash
   # 检查 21345 端口是否被占用
   lsof -i :21345

   # 修改 docker-compose.yml 中的端口映射
   ```

3. **数据库连接失败**
   ```bash
   # 检查 PostgreSQL 是否就绪
   docker-compose logs db | grep "ready to accept connections"
   ```

### 5.2 前端无法访问后端 API

**排查步骤**：

1. **检查 Nginx 配置**
   ```bash
   docker-compose exec frontend cat /etc/nginx/conf.d/default.conf
   ```

2. **验证反向代理**
   ```bash
   # 测试 /api/ 路径
   curl http://localhost:21345/api/health
   ```

3. **查看网络配置**
   ```bash
   docker-compose exec frontend ping backend
   ```

## 6. 性能问题

### 6.1 页面加载缓慢

**优化建议**：

1. **启用 Redis 缓存**
   - 确认 Redis 正常运行
   - 检查 `settings.py` 中的 CACHES 配置

2. **数据库索引**
   ```bash
   # 运行迁移确保索引创建
   docker-compose exec backend python manage.py migrate
   ```

3. **减少并发请求**
   - 前端避免同时请求大量基金数据
   - 使用分页加载

### 6.2 Celery 任务堆积

**排查步骤**：

1. **查看任务队列长度**
   ```bash
   docker-compose exec backend python -c "import redis; r=redis.Redis(host='redis'); print(r.llen('celery'))"
   ```

2. **增加 Worker 数量**
   ```yaml
   # docker-compose.yml
   worker:
     command: celery -A fundval worker --loglevel=info --concurrency=4
   ```

3. **清空失败任务**
   ```bash
   docker-compose exec backend celery -A fundval purge
   ```

## 7. 数据库问题

### 7.1 迁移失败

**解决方案**：

```bash
# 查看迁移状态
docker-compose exec backend python manage.py showmigrations

# 回滚迁移
docker-compose exec backend python manage.py migrate api <migration_name>

# 重新迁移
docker-compose exec backend python manage.py migrate
```

### 7.2 数据库损坏

**恢复步骤**：

```bash
# 备份当前数据
docker-compose exec db pg_dump -U fundval fundval > backup.sql

# 重建数据库
docker-compose down -v
docker-compose up -d db
docker-compose exec backend python manage.py migrate
docker-compose exec backend python manage.py sync_funds
```

## 9. 常见错误码

| 错误码 | 说明 | 解决方案 |
|--------|------|----------|
| 401 | 未授权 | 重新登录 |
| 403 | 权限不足 | 检查用户权限 |
| 404 | 资源不存在 | 确认 URL 和资源 ID |
| 500 | 服务器内部错误 | 查看后端日志 |
| 502 | 网关错误 | 检查后端服务是否启动 |
| 503 | 服务不可用 | 检查数据库/Redis 连接 |
